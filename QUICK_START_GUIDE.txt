STEP 1: OPEN FILE
   └─ Right-click on ams_aiops_dashboard.html
   └─ Select "Open with..." → Your Browser
   └─ File will open immediately

STEP 2: CREATE AN INCIDENT
   └─ Tab 1: "Incident Management" (selected by default)
   └─ Enter Incident Title: "Database response slow"
   └─ Enter Description: "Response time increased 300%"
   └─ Select Severity: "High"
   └─ Click "Analyze Incident"

STEP 3: VIEW RESULTS
   └─ Category will show: "DATABASE"
   └─ Confidence: 87% (example)
   └─ Root Cause: "Connection pool exhaustion"
   └─ Resolution: "Restart DB connection pool"
   └─ Runbook: "DB_RUNBOOK_001"
   └─ Click "Confirm Execution" to save

🆘 TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PROBLEM: Page shows blank
SOLUTION: Refresh page (F5) or clear browser cache (Ctrl+Shift+Delete)

PROBLEM: Data not showing in history
SOLUTION: This is normal on first use. Create an incident first, then history will show.

PROBLEM: "No incidents analyzed yet"
SOLUTION: Create an incident by filling the form and clicking "Analyze Incident"

PROBLEM: Styles look wrong
SOLUTION: Use a modern browser (Chrome, Firefox, Safari, Edge)

PROBLEM: Can't submit feedback
SOLUTION: Select an option from dropdown before clicking submit
